About

This project designs and implements a secure smartphone-based digital car key system aimed at car rental and fleet management. The system replaces traditional physical keys with securely provisioned digital keys, improving convenience and reducing the security risks associated with handling physical keys.

The prototype consists of a mobile Car Key App, a Vehicle Access Control Unit (VACU) running on a PC, a cloud-based management platform, and a CAN interface using Vector CANoe connected to a Jaguar Land Rover testing rig.

System Components

Car Key App

An Android app that authenticates the user via biometrics (fingerprint, Face ID) or PIN. Once authenticated, the app issues a signed lock or unlock command over Bluetooth Low Energy (BLE) to the VACU. Private keys are stored securely in the Android Keystore.

Vehicle Access Control Unit (VACU)

A Python application that acts as the BLE server and the secure control point between the mobile app and the vehicle network. It validates incoming requests using:

  • Cryptographic key pair checking
  • Timestamp and nonce verification
  • User authorisation rules

CANoe & JLR Test Rig

Once the VACU approves a request, it forwards the command to Vector CANoe via the Microsoft COM interface. CANoe generates and transmits a real CAN message to a Jaguar Land Rover testing rig, triggering simulated door lock and unlock events.

Cloud Backend & Manager App

The cloud backend handles user authentication, digital key provisioning, and event logging. Fleet managers use the Manager App to:

  • Create, assign, and revoke digital keys
  • Manage temporary guest access
  • Monitor vehicle access activity and logs

Cybersecurity

A key focus of this project is evaluating the security of a digital car key system. A threat model was developed covering the following attack types:

  • Replay Attacks - An attacker captures and replays a valid BLE unlock message. Mitigated using nonces and timestamps in a challenge-response protocol.
  • Man-in-the-Middle Attacks - An attacker intercepts the BLE transmission. Mitigated using BLE Secure Connections with pairing and bonding, and the VACU rejecting unsigned or altered messages.
  • Unauthorised Access - An attacker with a stolen KeyID attempts access. Mitigated by biometric/PIN authentication, secure keystore storage, and key revocation.
  • Cloud Data Breach - The cloud backend is compromised. Mitigated because private keys are stored only on the user's device, and TLS encryption is used for all communications.

Each attack vector will be tested in a controlled environment to demonstrate the system's resilience.
